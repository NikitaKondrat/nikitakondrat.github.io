## 1. Выполнение заданий

**Цель работы:** Освоить методы предобработки текстовых данных и векторизации для задачи классификации тональности на примере русскоязычных твитов.

**Задача:** Построить модель машинного обучения, которая по тексту твита предсказывает его эмоциональную окраску: положительную (`positive`) или отрицательную (`negative`).

#### Используемые методы

В работе были применены следующие техники обработки текста:

1. **Векторизация текста** — CountVectorizer, TfidfVectorizer
2. **Работа с n-граммами** — униграммы, биграммы, триграммы, пентаграммы
3. **Токенизация** — `word_tokenize` из библиотеки NLTK
4. **Фильтрация шума** — удаление стоп-слов и пунктуации
5. **Классификация** — LogisticRegression, XGBClassifier, LinearSVC

## 5. Улучшение предобработки: токенизация и стоп-слова

#### Настройка векторизатора

```
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from string import punctuation

# Загрузка ресурсов NLTK
nltk.download('punkt_tab')
nltk.download('stopwords')

# Список стоп-слов + пунктуация
noise = stopwords.words('russian') + list(punctuation)

# "Умный" векторизатор
smart_tokenizer = CountVectorizer(
    ngram_range=(1, 1), 
    tokenizer=word_tokenize,
    stop_words=noise
)
```

```
weighted avg       0.78      0.78      0.78     56709
```

**Улучшения:**

- Accuracy вырос с **0.76** до **0.78**

- Recall для негативного класса увеличился с 0.77 до **0.80**

## 7. Исследовательский анализ: роль пунктуации

#### Эксперимент: сохранение пунктуации

При отключении фильтрации стоп-слов и пунктуации:

```
vect_no_filter = CountVectorizer(ngram_range=(1,1), tokenizer=word_tokenize, stop_words=None)
```

**Результат:**

```
              precision    recall  f1-score   support

    negative       1.00      1.00      1.00     27959
    positive       1.00      1.00      1.00     28750

    accuracy                           1.00     56709
```

#### Анализ причины

Эмоциональные смайлики являются сильными индикаторами тональности:

```
# Простое правило на основе смайликов
pred_rule = ['positive' if ':D' in txt or ':)' in txt else 'negative' for txt in x_test]
```

**Результаты правила:**

| Метрика | Значение |
|---------|----------|
| Accuracy | 0.66 |
| Recall (negative) | 1.00 |
| Precision (positive) | 1.00 |

> **Важный вывод:** Пунктуация и эмодзи не всегда являются "шумом". В задачах анализа тональности они несут критически важную семантическую информацию. Решение об их удалении должно приниматься с учётом специфики задачи.

## 8. Символьные n-граммы

#### Эксперимент с character-level векторизацией

```
char_vec = CountVectorizer(analyzer='char', ngram_range=(1, 3))
```

**Результат:**

```
              precision    recall  f1-score   support

    negative       1.00      1.00      1.00     27959
    positive       1.00      1.00      1.00     28750

    accuracy                           1.00     56709
```

#### Объяснение

Символьные n-граммы захватывают последовательности, из которых состоят смайлики (`:)`, `:D`, `:(`), что позволяет модели идеально разделять классы.

**Преимущества символьных признаков:**

- Не требуют токенизации и лемматизации

- Работают для любых языков, включая те, у которых нет готовых анализаторов

- Эффективны для задач определения языка и детекции паттернов

**Недостатки:**

- Высокая размерность признакового пространства

- Риск переобучения на специфичных паттернах датасета
